入口文件

在 MVC 模式下，所有的请求都会发送到一个特殊的 PHP 脚本文件。这个文件被称为“入口”。新建好的应用程序在 public 目录或者根目录中有一个名为 index.php 的文件，这个文件就是“入口”。

“入口”的代码很少：

// 获得应用程序启动信息
$app_config = require(dirname(__FILE__) . '/../config/boot.php');
 
// 根据启动信息中的设置载入 QeePHP 框架和应用程序对象
require $app_config['QEEPHP_DIR'] . '/library/q.php';
require $app_config['APP_DIR'] . '/myapp.php';
 
// 构造应用程序对象，并启动 MVC 模式
MyApp::instance($app_config)->dispatching();

载入文件

入口文件首先会载入 boot.php 文件。boot.php 文件在应用程序的 config 目录中，包含了应用程序运行所需的基本信息。

通过 boot.php 中提供的信息，index.php 接下来会载入 QeePHP 框架的核心文件 q.php，以及应用程序对象文件 app/myapp.php。

至此，载入过程就结束了。

启动应用程序

由于应用程序对象被实现为单子模式，因此必须用 MyApp::instance() 的调用形式来获得一个应用程序对象实例。在构造实例时，需要传入从 boot.php 中获得的应用程序基本信息。这样应用程序对象才能完成后续的初始化工作。

最后，调用 dispatching() 方法对请求进行调度。这会导致某个控制器被调用，从而获得处理结果。
