多语言显示模型插件

作者：firzen

  • 分类：模型行为插件
  • 最后更新：2009-04-11 01:48
  • 下载次数：447
  • 项目网址：/projects/mui_language_display
  • 演示网址：http://qeephp.com/bbs/thread-6840-1-1.html
  • 在线文档：http://qeephp.com/bbs/thread-6840-1-1.html

在不需要修改view 和模型的基础上实现多语言显示。

例如：

$good->name
在中文语言环境下是：小衬衫
在英文语言环境下是：Little shirt

使用例子

模型 Good (Good表)

字段：gid , name , desc

模型 Good_Language (Good_lang 表)

字段：lid , lang_code , gid , name , desc

语言表字段需要和主表字段对应。如果 语言表没有指定语言的记录，返回主表记录。

使用配置：

'behaviors' => 'language',
'behaviors_settings' => array
(
'language' => array(
    'lang_class' => 'Good_Language',   // 这里是 Good_lang表对应的model
    'lang_fields' => 'name,desc',                   // 有语言属性的字段
    'language' => Q::ini('app_config/language'),    // 对应 lang_code
    'target_key' => 'gid'                                //与主表关联键
),

使用（无需修改）：

$good=Good::find()->getOne();
echo $good->name ; // 自动判断多语言

注意事项

如果你的属性设置的了getter ，请改为外部调用方式。

例如：

我的 $good->property 属性设置为 array(getter=>getProperty)，

那么需要修改为 array(getter=>array(Good,getProperty));
